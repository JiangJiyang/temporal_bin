# zch 工程协作说明

## 当前包的定位

这是给“普通 ChatGPT 协作修改工程”准备的单包版本，不依赖 agent 分包恢复。

## 当前环境假设

- 公网 Qwen3-235B int8 网关：由用户自行配置，可作为主质量增强链路。
- MinIO：当前不可用或暂不作为主依赖。
- 本地测试数据主路径：`./测试项目`
- 批量输入文件：`./filejsonrst.json`
- Prompt 文件：`./prompt.json`

## 当前包已做的预处理

1. 已修复原始压缩包中的中文目录转义问题，`测试项目` 及其子路径可按正常中文路径访问。
2. 已将工程默认行为调整为“本地测试集优先、MinIO 不阻塞”。
3. `config.json` 中的公网模型和 MinIO 已改成占位模板，避免误用旧配置或泄露敏感信息。

## 使用建议

1. 若你要实际运行模型调用，请先把 `config.json` 中 `api_model.base_url` 与 `api_model.api_key` 改成你自己的有效 Qwen 网关配置。
2. 若 MinIO 仍不可用，不应阻塞开发、测试、报告与交付；应默认基于本地 `测试项目` 完成工作。
3. 若运行环境无法访问公网模型，应自动降级到本地模型或无模型规则模式，而不是直接停止工程。

## 关键文件

- `main.py`：统一核心逻辑入口
- `fastapi_app.py`：FastAPI 入口
- `config.json`：运行配置
- `filejsonrst.json`：批量测试输入
- `prompt.json`：LLM 抽取提示模板
- `测试项目/`：本地测试数据
